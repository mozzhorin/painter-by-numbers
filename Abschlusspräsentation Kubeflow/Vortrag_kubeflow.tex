\textbf{What is Kubeflow?}
\begin{itemize}
\item an open-source platform \\
\item based on Kubernetes \\
\item simple, portable and scalable deployments of ML \\
\item integration of existing tools and libraries
\end{itemize}
}

\frame{
\textbf{Use Case}
\begin{itemize}
\item Train a CNN model to distinguish between works of 10 painters\\
\item Dataset from kaggle: https://www.kaggle.com/c/painter-by-numbers/\\
Images with their information about artist, date, genre, style and title
\item Size of Dataset:  (after Resizing: )\\
\item Objective: A model that gives a (good) predicition who painted the incoming image
\end{itemize}
}

\frame{
\textbf{Milestones}
\begin{itemize}
\item Data preprocessing
\item Create a model (transfer learning)\\
\item Create a Docker image\\
\item Train the model on GKE using TFJobs on CPUs\\
\item Train the model on GKE using TFJobs on GPUs\\
\item Distributed training on multiple pods\\
\item Save the trained model into a GCS bucket\\
\item Serve the model with TFServing
\end{itemize}
}

\frame{
\textbf{Technologies}
\begin{itemize}
\item TF for the model $\rightarrow$ keras\\
\item Docker + Kubernetes\\
\item Kubectl + ksonnet $\rightarrow$ kubectl + kustomize $\rightarrow$b kubectl\\
\item GCP dashboard/console + gcutil\\
\item TF Serving: Tensorboard
\end{itemize}
}

\textbf{Conclusion}
\begin{itemize}
\item Requirement of understanding the used technologies (Dockers, Kubernetes, Keras...)\\
\item still a raw version (ksonnet,...)
\item Kubeflow is not for data science - it's for model training
\end{itemize}
